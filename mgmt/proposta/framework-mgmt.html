Nossa leitura

Onde a MGMT está hoje

Produto em produção desde nov/2024, base de usuários ativa e clientes pagantes — a tese da MGMT já se provou no mercado.

Stack em Python sobre Amazon Bedrock, LangChain 0.14, arquitetura conversacional como primeira camada de atendimento financeiro. Tudo funciona — mas evoluir o sistema virou desafio maior do que construir do zero.

O motivo é simples: não existe rede de segurança. Testar LLM não se comporta como TDD tradicional, migrar versão de framework sem evals é salto no escuro, e toda a stack depende de uma pessoa só.

Como a Taqtile entra

Framework em três pilares

A Taqtile aloca um engenheiro sênior part-time como parceiro técnico da MGMT. A ordem dos pilares não é arbitrária — cada fase depende da anterior funcionando.

01
Fase 1 · 4-6 semanas

Rede de proteção

Framework de evals cobrindo cenários críticos do agente MGMT. Sem essa rede, tudo que vier depois é aposta.

  • Suíte de testes rodando em CI/CD com gates por threshold
  • Bateria semanal de 10-20-100 iterações para métricas estatísticas
  • Integração com Amazon Bedrock respeitando compliance regulatório
  • MGMT co-autora do framework — não recebe, constrói junto
02
Fase 2 · 8-12 semanas

Migração controlada

Reescrita LangChain 0.14 → 1.10 cadeia-a-cadeia com a rede da Fase 1 ativa. Ataque específico ao problema de drift de memória.

  • Mapeamento de dependências + prototipagem 1.10 antes de escalar
  • Merge bloqueado automaticamente se eval falha
  • Checkpointer customizado e API reset-to-message
  • Rollout controlado (subset de usuários) antes de 100%
03
Fase 3 · 2-3 semanas

Autonomia como meta

Hand-off é fase formal, não promessa vaga. Sucesso da parceria é a MGMT operando o framework sozinha com segurança.

  • Documentação operacional pensada para leitor solo
  • Treinamento prático — MGMT opera, Taqtile observa
  • Runbooks para cenários críticos (Bedrock indisponível, drift, etc.)
  • Check-in mensal opcional por 3 meses após saída
